This dashboard reads two files, relative to dashboard/:

  • ../outreach_log.jsonl — one JSON object per line, one line per lead, written automatically by the pipeline when a lead is found and updated by tracker/update.js as you send/hear back.
  • ../tracker/clicks.jsonl — written automatically by the tracking server when a recipient opens a tracking_url.

Example outreach_log.jsonl line:

{"lead_id":"acme-inc-x7q2","date_found":"2026-07-14T10:00:00.000Z","company_name":"Acme Inc","contact_or_repo_owner":"alexdev","signal_type":"github","evidence":"llm_router.py hand-rolls OpenAI/Anthropic failover","fit_score":9,"variant":"A","tracking_url":"http://localhost:3000/r/acme-inc-x7q2","message_draft":"Hi Alex, ...","date_sent":null,"replied":null,"signed_up":null}

Serve this directory over HTTP (fetch of local files is blocked over file://) — e.g. from the repo root: python3 -m http.server 8080, then open http://localhost:8080/dashboard/.
